EXCEL DATA TRANSFORMATION TUTORIAL
How to Transform Test Cases CSV Data in Excel Using Data Tab
================================================================

STEP 1: OPEN THE CSV FILE IN EXCEL
-----------------------------------

1. Open Microsoft Excel
2. Click File menu, then Open
3. Navigate to test-cases-piffdeals-staff.csv
4. Excel will show a Text Import Wizard
5. Select "Delimited" option
6. Click Next
7. Select "Semicolon" as delimiter (since CSV uses semicolons)
8. Click Next
9. Click Finish

The CSV will open with all columns properly separated.


STEP 2: UNDERSTAND THE COLUMN STRUCTURE
----------------------------------------

Your CSV has these columns:
- Column A: Test ID
- Column B: Test Case
- Column C: Category
- Column D: Subcategory
- Column E: Input Data
- Column F: Expected Output
- Column G: Test Steps
- Column H: Result (currently empty)
- Column I: Notes (currently empty)
- Column J: Column1 (currently empty)
- Column K: _1 (currently empty)


STEP 3: USING DATA TAB FOR TRANSFORMATIONS
-------------------------------------------

Go to the Data tab in Excel ribbon. Here are common transformations:


A. TEXT TO COLUMNS (Split Data)
--------------------------------

If you need to split a column into multiple columns:

1. Select the column you want to split (e.g., Column B - Test Case)
2. Go to Data tab
3. Click "Text to Columns" button
4. Choose "Delimited" or "Fixed width"
5. Select delimiter (space, comma, semicolon, etc.)
6. Click Next
7. Preview the split, adjust if needed
8. Click Finish

Example: If Test Case column contains "Login with correct credentials", you could split it by "with" to get two columns.


B. REMOVE DUPLICATES
--------------------

To find and remove duplicate test cases:

1. Select all data (Ctrl+A)
2. Go to Data tab
3. Click "Remove Duplicates" button
4. Select which columns to check for duplicates
5. Click OK
6. Excel will show how many duplicates were removed


C. SORT DATA
------------

To sort test cases by any column:

1. Select all data including headers
2. Go to Data tab
3. Click "Sort" button
4. Choose column to sort by (e.g., Test ID, Category, etc.)
5. Choose sort order (A to Z, Z to A, or custom)
6. Add multiple sort levels if needed
7. Click OK


D. FILTER DATA
--------------

To filter and view specific test cases:

1. Select header row
2. Go to Data tab
3. Click "Filter" button (funnel icon)
4. Dropdown arrows appear in each column header
5. Click dropdown to filter by:
   - Category (Frontend/Backend)
   - Subcategory
   - Status (if you fill Result column)
   - Any other column

Example: Filter to show only "Frontend" test cases, or only "Invoices" subcategory.


E. FIND AND REPLACE
-------------------

To replace text across all cells:

1. Press Ctrl+H (or go to Home tab, Find & Select, Replace)
2. Enter text to find (e.g., "Frontend")
3. Enter replacement text (e.g., "Front-end")
4. Choose to replace all or one by one
5. Click Replace All or Replace

Useful for fixing typos or standardizing terminology.


F. DATA VALIDATION
-------------------

To create dropdown lists in cells (e.g., for Result column):

1. Select column where you want dropdown (e.g., Column H - Result)
2. Go to Data tab
3. Click "Data Validation" button
4. In Settings tab:
   - Allow: List
   - Source: Type values separated by commas (e.g., Pass, Fail, Pending, Not Tested)
5. Click OK

Now cells in that column will have dropdown arrows with your options.


G. CONSOLIDATE DATA
-------------------

To combine data from multiple ranges:

1. Go to Data tab
2. Click "Consolidate" button
3. Choose function (Sum, Count, Average, etc.)
4. Select ranges to consolidate
5. Click OK

Useful if you have test results in separate sheets.


H. GROUP AND OUTLINE
---------------------

To group rows by category for easier navigation:

1. Select rows you want to group (e.g., all Frontend test cases)
2. Go to Data tab
3. Click "Group" button
4. Choose "Rows" or "Columns"
5. Click OK

You can collapse/expand groups using +/- buttons.


STEP 4: FILLING EMPTY COLUMNS FOR NEW TEST CASES
--------------------------------------------------

For the new test cases (301-360), you may want to fill empty columns:


A. FILL RESULT COLUMN (Column H)
---------------------------------

1. Select Column H header
2. Add Data Validation dropdown with: Pass, Fail, Pending, Not Tested, Blocked
3. For each test case, select appropriate result from dropdown
4. Or type directly: Pass, Fail, etc.


B. FILL NOTES COLUMN (Column I)
--------------------------------

1. Click in Column I cells
2. Type notes about test execution:
   - "Tested on Chrome browser"
   - "Requires admin access"
   - "Needs test data setup"
   - "Automated test available"
   - Any relevant information


C. REMOVE UNUSED COLUMNS (Column J and K)
------------------------------------------

If Column1 and _1 are not needed:

1. Right-click on Column J header
2. Select "Delete"
3. Repeat for Column K
4. Or hide them: Right-click, Hide


STEP 5: FORMATTING THE DATA
----------------------------

A. FORMAT AS TABLE
------------------

1. Select all data
2. Go to Home tab
3. Click "Format as Table"
4. Choose a table style
5. Check "My table has headers"
6. Click OK

Benefits:
- Automatic filtering
- Alternating row colors
- Easy sorting
- Professional appearance


B. FREEZE PANES
---------------

To keep header row visible while scrolling:

1. Click on row 2 (first data row)
2. Go to View tab
3. Click "Freeze Panes"
4. Select "Freeze Panes"

Now header row stays visible when scrolling down.


C. CONDITIONAL FORMATTING
-------------------------

To highlight test cases based on criteria:

1. Select data range (e.g., all Result column cells)
2. Go to Home tab
3. Click "Conditional Formatting"
4. Choose rule type:
   - Highlight cells with "Pass" in green
   - Highlight cells with "Fail" in red
   - Color code by Category
5. Set formatting options
6. Click OK

Example: Format all "Backend" test cases with blue background.


STEP 6: EXPORTING TRANSFORMED DATA
-----------------------------------

After making transformations:

1. Go to File menu
2. Click "Save As"
3. Choose file format:
   - CSV (Comma delimited) - for simple CSV
   - CSV UTF-8 - for international characters
   - Excel Workbook (.xlsx) - to keep formatting
4. Enter filename
5. Click Save

If saving as CSV, Excel will warn about losing formatting - click Yes if you want plain CSV.


STEP 7: COMMON TRANSFORMATION SCENARIOS
----------------------------------------

SCENARIO 1: Split Test Case column into multiple parts
-------------------------------------------------------
1. Select Column B (Test Case)
2. Data tab > Text to Columns
3. Delimited > Space
4. This splits "Login with correct credentials" into separate columns
5. Adjust as needed


SCENARIO 2: Add status to all new test cases
----------------------------------------------
1. Select Column H (Result) for rows 302-361
2. Type "Not Tested" or use Data Validation dropdown
3. Press Ctrl+Enter to fill all selected cells at once


SCENARIO 3: Filter to see only new test cases
----------------------------------------------
1. Click Filter button (Data tab)
2. Click dropdown on Test ID column
3. Choose "Number Filters" > "Greater Than"
4. Enter 300
5. Click OK
6. Now only test cases 301-360 are visible


SCENARIO 4: Count test cases by category
------------------------------------------
1. Select all data
2. Insert tab > PivotTable
3. Drag "Category" to Rows
4. Drag "Test ID" to Values (set to Count)
5. See count of Frontend vs Backend test cases


SCENARIO 5: Create summary sheet
---------------------------------
1. Right-click sheet tab > Insert > Worksheet
2. Name it "Summary"
3. Use formulas:
   - Total tests: =COUNTA(Sheet1!A:A)-1
   - Frontend tests: =COUNTIF(Sheet1!C:C,"Frontend")
   - Backend tests: =COUNTIF(Sheet1!C:C,"Backend")
   - New tests (301-360): =COUNTIFS(Sheet1!A:A,">=301",Sheet1!A:A,"<=360")


STEP 8: TIPS AND BEST PRACTICES
--------------------------------

1. Always work on a copy of original CSV file
2. Use Excel's Undo (Ctrl+Z) if you make mistakes
3. Save frequently (Ctrl+S)
4. Use Freeze Panes to keep headers visible
5. Format as Table for better data management
6. Use Data Validation to ensure consistent data entry
7. Add filters to quickly find specific test cases
8. Use conditional formatting to visualize test status
9. Create backup before major transformations
10. Document any manual changes in Notes column

STEP 10: TROUBLESHOOTING
-------------------------

PROBLEM: CSV opens with all data in one column
SOLUTION: Use Text to Columns with semicolon delimiter

PROBLEM: Numbers showing as text
SOLUTION: Select column > Data tab > Text to Columns > Finish (forces conversion)

PROBLEM: Lost formatting after saving as CSV
SOLUTION: This is normal - CSV doesn't support formatting. Save as .xlsx to keep formatting.

PROBLEM: Filter not working
SOLUTION: Make sure header row is included when applying filter

PROBLEM: Formulas not calculating
SOLUTION: Check if calculation is set to Manual (Formulas tab > Calculation Options > Automatic)
